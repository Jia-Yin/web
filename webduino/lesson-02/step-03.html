# 3 專案檔的管理

我們發現法蘭斯的專案檔很多，而且也可以分享給其他人，那麼我們的專案檔可能也會越來越多，也許也需要和別人分享，應該要放哪裡，怎麼管理呢？以下簡單說明一下 Webduino 管理平台的基本使用。

  1. 首先，我們目前大多使用 Webduino 的開發平台，最方便的地方可能是放在這個平台上。處理的方式也很簡單，我們先看看進開發平台的首頁： 這邊可以看到，左邊有「我的檔案」、「共享的檔案」、「公開的檔案」三類，目前我們的檔案都放在「我的檔案」中。在上方的圖示那排的最右邊，有一個 5/20 的數字，表示我們目前可以存 20 個專案，而現在已經用了 5 個。「共享的檔案」，應該是別人分享給我們的檔案，一般情況下，如果沒有人分享專案給我們，點選進去應該是空的。另外「公開的檔案」點進去，會看到一個搜尋框，如果我們在上面輸入一些搜尋的關鍵字，例如「超音波」，就會跑出很多公開分享的專案出來。

  2. 我們也可以把檔案分享給別人，或者公開到網路上。方法也很簡單，先勾選一個專案檔，然後按上面分享的那個圖示（三個點兩條線那個），就會出現以下的畫面：

    這邊選「指定使用者」的話，可以輸入想要分享對象的 Email 帳號，那麼對方就可以取用我們的專案了。選「公開」的話，別人就可以利用名稱的關鍵字搜尋到我們的專案。最後一個「非公開」，就是只要有專案的連結，就可以讀取，但無法利用關鍵字搜尋的方式找到這個專案。

  3. 購買 Webduino 套件時，如果有拿到兌換卷的話，可以在專案畫面右上方人頭旁邊那個像迴紋針的按鈕按一下，

    就會出現一個輸入框，讓我們可以填入序號，正確填入之後，我們可以放的專案數就會變多了。

  4. 如果專案數很多，給定的數量不夠用，那其實 Webduino 平台也提供了檔案匯入和匯出的功能。在開發平台的左下方，有一個像齒輪一樣的按鈕，參照下圖：

    點選那個齒輪鈕之後，會出現以下的畫面：

    在這個圖中，下面是匯出的按鈕，可以下載目前的設計，並存檔到自己的電腦中。而上面則是匯入的按鈕，可以選擇從檔案匯入，或者從連結匯入。從檔案匯入的話，可以把之前存的檔案恢復回來，而從連結匯入，只要填入一個有效的專案連結（在使用體驗版設計平台的時候，有一個按鈕可以產生連結），這樣就可以把該連結的設計專案匯到平台裡了。使用這個匯入匯出的功能，再多的專案也不怕了。

  5. 除了 Webduino 平台之外，我們也可以把專案放在 jsbin.com、jsFiddle、Github、Gitlab... 等各種網頁開發平台，或靜態網頁托管區，這些地方基本上都沒有專案數的限制。在我們使用 Webduino 開發平台的時候，有一個可以看程式碼的按鈕，我們發現它會進到 jsbin 的網站中，所以這也是最方便的一個地方，只要申請 Github 帳號，就可以使用 jsbin.com，然後只要把 Webduino 產生的程式碼複製到我們的專案中，這樣我們就可以保存專案並分享成果，而且針對同一個專案還可以保留許多不同的版本喔。這部份需要懂得一些進階的操作方法，有興趣的讀者可以花些時間研究看看。


練習

  1. 試著使用 Webduino 平台上的分享功能，用不同方式把自己的專案分享給其他人。練習完之後，如果不想分享的話，可以把它改回來。
  2. 試著使用 jsbin.com，把 Webduino 的專案存在我們自己的 jsbin.com 帳號中。
